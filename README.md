# منصة إدارة العملاء المحتملين — UIS

منصة متكاملة بـ 3 واجهات: **الإدارة**، **السيلز**، **التيكنيكال سبورت**، شغالة على قاعدة بيانات واحدة (Supabase) ومستضافة مجانًا على Vercel.

بيانات مشروعك على Supabase متحطة بالفعل جوه `assets/app.js` — الملفات دي جاهزة للنشر مباشرة.

## هيكل المشروع
```
index.html          → بيوجّه أوتوماتيك لـ login.html
login.html          → تسجيل الدخول، بيوجّه كل حد لواجهته حسب دوره
sales.html           → Kanban للعملاء + نموذج اكتشاف + بيانات مالية + عقد + نشاط
support.html         → مراجعة كل النماذج + Debrief + مرحلة التنفيذ
management.html      → لوحة KPIs + Charts + تارجت + تجديدات + مصادر عملاء + نشاط الفريق
report.html          → تقرير التنفيذ والـ Go-Live لكل عميل (قابل للطباعة PDF)
assets/styles.css     → التصميم (أزرق/أبيض، محدّث)
assets/app.js         → الاتصال بـ Supabase + دوال مشتركة + بيانات مشروعك
schema.sql .. schema_v7.sql → كل تحديثات قاعدة البيانات بالترتيب
```

## خطوات التشغيل من الصفر (رتّبها بالظبط كده)

### ١. قاعدة البيانات
روح لمشروعك في supabase.com → **SQL Editor → New query**، وشغّل الملفات دي **بالترتيب** (كل ملف لوحده، Run، وبعدين اللي بعده):

1. `schema.sql` — الجداول الأساسية (profiles, clients, debriefs) + تسجيل الدخول والأدوار
2. `schema_v2.sql` — تقارير التنفيذ والـ Go-Live
3. `schema_v3.sql` — الماليات والتوقعات (قيمة الصفقة، التارجت)
4. `schema_v5.sql` — مرحلة التنفيذ بعد كسب الصفقة (تجاهل `schema_v4.sql` — كانت فكرة تذاكر دعم، مش محتاجينها لأن عندكم نظام تذاكر منفصل)
5. `schema_v6.sql` — سجل نشاط الفريق (زيارات/مكالمات/متابعات)
6. `schema_v7.sql` — مصادر العملاء (Lead Source)

### ٢. إنشاء حسابات الموظفين وتحديد الأدوار
1. **Authentication → Users → Add user** لكل موظف (إيميل + باسورد)
2. كل حساب جديد بياخد دور "sales" تلقائي (عن طريق trigger في `schema.sql`)
3. عشان تغيّر حد لـ "admin" أو "support": **Table Editor → profiles** → غيّر عمود `role`
4. عدّل `full_name` لكل شخص من نفس الشاشة عشان اسمه يبان صح في الواجهة

### ٣. الرفع على Vercel
ارفع مجلد المشروع كامل (بكل الملفات والمجلد `assets/`) على vercel.com — بالسحب المباشر أو عن طريق GitHub. هتاخد لينك زي `your-project.vercel.app`.

## كيف يشتغل الفريق
- **السيلز**: يدخل بحسابه، يشوف عملاءه بس على Kanban، يضيف عميل جديد بنموذج الاكتشاف (فيه دلوقتي: القيمة المالية، مصدر العميل، بيانات العقد)، يسجّل نشاطه (زيارات/مكالمات)، ويحرّك الكارت بين المراحل
- **التيكنيكال سبورت**: يشوف كل النماذج من كل السيلز، يراجعها، يضيف Debrief، ويحدّث مرحلة التنفيذ لأي عميل "فوز"
- **الإدارة**: تشوف كل الأرقام في مكان واحد — الـ Pipeline، الماليات، التارجت مقابل الفعلي، العقود القريبة من التجديد، مصادر العملاء الأفضل أداءً، ونشاط كل سيلز

## الحماية
كل الصلاحيات محكومة من قاعدة البيانات نفسها (Row Level Security)، مش بس من الواجهة. وكل نص بيكتبه أي مستخدم (اسم عميل، ملاحظات، إلخ) بيتم تنظيفه قبل عرضه في الصفحة لمنع أي محاولة حقن أكواد ضارة (XSS).

## مراجعة الكود (آخر تحديث)
راجعنا الكود القديم بالكامل قبل إضافة أي حاجة جديدة، ولقينا وصلحنا:
- باگ في `report.html` كان ممكن يوقّع الصفحة لو فتحت على عميل محذوف
- كل نص يكتبه المستخدم بقى بيتنضف (escapeHtml) قبل ما يتحط في الصفحة، عشان نقفل أي احتمال لـ XSS
- تأكدنا إن كل زرار في المنصة كلها مربوط بدالة موجودة فعلاً، مفيش زرار "ميت"

## التصميم
حدّثنا `assets/styles.css` بالكامل: انتقالات وحركات أنعم، حالة تحميل (Spinner)، تأثيرات hover على الكروت والأزرار والجداول، ظهور أسهل للـ Modals، وتباين ألوان أوضح — نفس الهوية الزرقاء/البيضاء بس بشكل أحدث وأسهل استخدام.
