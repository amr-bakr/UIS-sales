# منصة إدارة العملاء المحتملين — UIS

منصة متكاملة بـ 3 واجهات: **الإدارة**، **السيلز**، **التيكنيكال سبورت**، شغالة على قاعدة بيانات واحدة (Supabase) ومستضافة مجانًا على Vercel.

## هيكل المشروع
```
index.html          → بيوجّه أوتوماتيك لـ login.html
login.html          → تسجيل الدخول، بيوجّه كل حد لواجهته حسب دوره
sales.html           → Kanban للعملاء بتاعة كل سيلز + نموذج اكتشاف جديد
support.html         → قايمة مراجعة كل النماذج من كل السيلز + Debrief
management.html      → لوحة KPIs + Chart + أداء كل سيلز
assets/styles.css     → التصميم (أزرق/أبيض)
assets/app.js         → الاتصال بـ Supabase + دوال مشتركة
schema.sql            → كود قاعدة البيانات الكامل
```

## ١. إنشاء مشروع Supabase (مجاني)
1. supabase.com → حساب مجاني → **New project**
2. من **SQL Editor → New query**، انسخ محتوى `schema.sql` بالكامل والصقه ودوس **Run**
3. من **Project Settings → API** خد **Project URL** و **anon public key**

## ٢. ربط الملفات بالمشروع
افتح `assets/app.js` وحط القيم بتاعتك بدل:
```js
const SUPABASE_URL = "https://YOUR-PROJECT.supabase.co";
const SUPABASE_ANON_KEY = "YOUR-ANON-KEY";
```

## ٣. إنشاء حسابات الموظفين وتحديد الأدوار
1. من **Authentication → Users → Add user**، أضف حساب لكل موظف (إيميل + باسورد)
2. بمجرد ما الحساب يتعمل، هيتضاف تلقائي صف ليه في جدول `profiles` بدور "sales" افتراضي
3. عشان تغيّر دور حد لـ "admin" أو "support": روح **Table Editor → profiles**، دوس على صف الشخص، وغيّر عمود `role` لـ `admin` أو `support` حسب دوره الفعلي
4. لو حابب تضيف اسمه الظاهر في الواجهة، عدّل عمود `full_name` من نفس الشاشة

## ٤. الرفع على Vercel (مجاني)
1. vercel.com → حساب مجاني → **Add New → Project**
2. ارفع مجلد المشروع كامل (فيه كل الملفات اللي فوق) — إما بالسحب المباشر أو عن طريق ربط GitHub repo
3. Vercel هيديك لينك زي `your-project.vercel.app`

## كيف يشتغل الفريق
- **السيلز**: يدخل بحسابه على نفس اللينك، يوصله على `sales.html`، يشوف عملاءه بس، يضيف عميل جديد، ويحرّك الكارت بين مراحل الـ Pipeline
- **التيكنيكال سبورت**: يدخل بحسابه، يوصله على `support.html`، يشوف كل النماذج من كل السيلز، يحدد "تم الاستلام" أو "يحتاج استكمال"، ويضيف Debrief بعد كل ديمو
- **الإدارة**: تدخل بحسابها، توصلها على `management.html`، تشوف الأرقام: إجمالي العملاء، نسبة التحويل، توزيع المراحل، وأداء كل سيلز

## الحماية
كل الصلاحيات دي متحكم فيها من قاعدة البيانات نفسها (Row Level Security)، مش بس من الواجهة — يعني حتى لو حد فتح كود الصفحة، مش هيقدر يشوف بيانات عميل مش بتاعه غير لو دوره admin أو support فعليًا في جدول profiles.
