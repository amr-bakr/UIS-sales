# خطوات التشغيل (Vercel + Supabase)

## ١. إنشاء مشروع Supabase (مجاني)
1. روح على supabase.com وسجّل حساب مجاني
2. "New project" → اختار اسم وباسورد لقاعدة البيانات (احفظه في مكان آمن)
3. استنى دقيقتين لحد ما المشروع يجهز

## ٢. إنشاء الجدول
1. من القائمة الجانبية: **SQL Editor** → **New query**
2. افتح ملف `schema.sql` المرفق، انسخ محتواه بالكامل، الصقه، ودوس **Run**
3. تأكد إن الجدول `entries` ظهر في **Table Editor**

## ٣. تفعيل حسابك أنت (تيكنيكال سبورت) بس
1. من القائمة الجانبية: **Authentication → Users → Add user**
2. حط إيميلك وكلمة مرور (ده اللي هتسجل بيه دخول في تبويب "مراجعة التيكنيكال سبورت")
3. ده الحساب الوحيد اللي هيقدر يشوف النماذج ويعدّل حالتها — السيلز مش محتاج يسجل دخول خالص عشان يبعت نموذج

## ٤. ربط الملف بالمشروع
1. من **Project Settings → API**، هتلاقي:
   - **Project URL**
   - **anon public key**
2. افتح ملف `index.html`، ولاقي السطرين دول فوق في الـ script:
   ```js
   const SUPABASE_URL = "https://YOUR-PROJECT.supabase.co";
   const SUPABASE_ANON_KEY = "YOUR-ANON-KEY";
   ```
3. استبدلهم بالقيم بتاعتك واحفظ الملف

## ٥. الرفع على Vercel (مجاني)
**أسهل طريقة (من غير حتى GitHub):**
1. روح على vercel.com وسجّل حساب مجاني
2. من الداشبورد دوس **Add New → Project**
3. اختار **"Deploy without Git"** أو اسحب مجلد فيه `index.html` بس على صفحة الرفع
4. Vercel هيديك لينك زي `your-project.vercel.app` — ده اللينك اللي تبعته للسيلز

**لو عايز تحديثات مستقبلية أسهل:** ارفع الملف على GitHub repo، واربط الـ repo بـ Vercel، وبعد كده أي تعديل تعمله وترفعه على GitHub بيتحدث على اللينك تلقائي.

## ملاحظات مهمة
- الحماية شغالة من جانب قاعدة البيانات نفسها (Row Level Security)، مش بس من الواجهة — يعني حتى لو حد فتح كود الصفحة وشاف الـ anon key، مش هيقدر يشوف بيانات العملاء من غير ما يسجل دخول بالإيميل والباسورد بتاعك.
- تقدر تضيف حسابات تانية لاحقًا من نفس شاشة Authentication لو حد تاني من التيكنيكال سبورت احتاج access.
- الباقة المجانية في Supabase كافية جدًا لحجم استخدام زي ده (آلاف الصفوف مجانًا).
